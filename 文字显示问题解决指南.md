# ERP 项目 - 文字显示问题（????）解决指南

**日期**：2026-04-02  
**问题**：界面文字显示为 "????"

---

## 🔍 问题分析

### 问题现象
界面菜单、按钮、标签等文字显示为 "????"

### 可能原因
1. 浏览器语言设置不正确
2. 前端语言包未正确加载
3. 前端开发服务器需要重启
4. 后端响应编码问题

---

## 🛠️ 解决方案

### 方案1：在浏览器中手动设置语言（最简单，推荐）

#### 步骤：
1. 在浏览器中按 **F12** 打开开发者工具
2. 点击 **Console（控制台）** 标签
3. 输入以下命令并回车：
   ```javascript
   localStorage.setItem('language', 'zh-CN')
   ```
4. 按 **F5** 刷新页面
5. 文字应该正常显示了！

---

### 方案2：修改浏览器语言设置

#### Chrome/Edge 浏览器：
1. 打开浏览器设置
2. 搜索 "语言"
3. 点击 "语言"
4. 点击 "添加语言"
5. 选择 "中文（简体）"
6. 将 "中文（简体）" 拖动到列表顶部
7. 重启浏览器
8. 刷新页面

---

### 方案3：重启前端开发服务器

#### 步骤：
1. 在运行前端的终端中按 **Ctrl + C** 停止前端
2. 重新启动前端：
   ```bash
   cd d:\erp\RuoYi-Vue\ruoyi-ui
   npm run dev
   ```
3. 等待前端重新启动
4. 刷新浏览器页面

---

### 方案4：检查并清除浏览器缓存

#### 步骤：
1. 按 **Ctrl + Shift + Delete** 打开清除浏览器数据
2. 选择 "缓存的图片和文件"
3. 点击 "清除数据"
4. 刷新页面

---

### 方案5：检查后端响应编码

#### 验证步骤：
1. 按 **F12** 打开开发者工具
2. 点击 **Network（网络）** 标签
3. 刷新页面
4. 点击任意请求
5. 查看 **Response Headers（响应头）**
6. 确认 `Content-Type` 包含 `charset=utf-8`
