## 三、迁移前准备

### 3.1 需要准备的工具

1. **MySQL** - 用于数据库导入导出
2. **JDK 17+** - 用于运行后端
3. **Node.js** - 用于运行前端
4. **Maven** - 用于构建后端项目
5. **Git**（可选）- 用于代码版本管理

### 3.2 需要迁移的内容

1. **数据库** - ry_vue 数据库
2. **后端代码** - d:\erp\RuoYi-Vue\ruoyi-admin
3. **前端代码** - d:\erp\RuoYi-Vue\ruoyi-ui
4. **配置文件** - 数据库连接配置等

## 四、详细迁移步骤

### 选项A：迁移到 Windows 11

#### 步骤A1：导出数据库（在当前PC上执行）

```powershell
# 使用命令行导出
mysqldump -u root -p ry_vue > ry_vue_backup_20260412.sql

# 或者使用Navicat/MySQL Workbench等工具导出（推荐）
```

#### 步骤A2：复制代码文件

将整个 `d:\erp\RuoYi-Vue` 文件夹复制到新PC，例如：
```
C:\erp\RuoYi-Vue\
├── ruoyi-admin\    # 后端代码
├── ruoyi-ui\       # 前端代码
└── pom.xml         # Maven配置
```

#### 步骤A3：安装环境（Windows 11）

1. **安装 MySQL 8.0+**
   - 下载地址：https://dev.mysql.com/downloads/mysql/
   - 安装完成后记住 root 密码

2. **安装 JDK 17+**
   - 下载地址：https://adoptium.net/
   - 选择 Windows x64 JDK 17
   - 安装后配置环境变量 JAVA_HOME

3. **安装 Node.js 16+**
   - 下载地址：https://nodejs.org/
   - 选择 LTS 版本
   - 安装后 npm 会自动添加到环境变量

4. **安装 Maven 3.6+**
   - 下载地址：https://maven.apache.org/download.cgi
   - 解压后配置环境变量 MAVEN_HOME

#### 步骤A4：导入数据库

```powershell
# 打开PowerShell，连接MySQL
mysql -u root -p
```

在MySQL命令行中执行：
```sql
CREATE DATABASE ry_vue DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
EXIT;
```

导入数据：
```powershell
mysql -u root -p ry_vue < C:\path\to\ry_vue_backup_20260412.sql
```

#### 步骤A5：配置数据库连接

修改配置文件：
```
C:\erp\RuoYi-Vue\ruoyi-admin\src\main\resources\application-druid.yml
```

更新数据库连接信息：
```yaml
url: jdbc:mysql://localhost:3306/ry_vue?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
username: root
password: [你的MySQL密码]
```

#### 步骤A6：编译并启动后端

```powershell
cd C:\erp\RuoYi-Vue

# 编译后端（可能需要几分钟）
mvn clean install -DskipTests

# 启动后端
cd ruoyi-admin
mvn spring-boot:run
```

后端将运行在 `http://localhost:8080`

#### 步骤A7：安装依赖并启动前端

```powershell
cd C:\erp\RuoYi-Vue\ruoyi-ui

# 安装依赖
npm install

# 如果安装慢，可以使用淘宝镜像
# npm install --registry=https://registry.npmmirror.com

# 启动前端
npm run dev
```

前端将运行在 `http://localhost:80`

### 选项B：迁移到 Linux（Ubuntu/CentOS）

#### 步骤B1：导出数据库（在当前PC上执行）

```bash
# 导出数据库
mysqldump -u root -p ry_vue > ry_vue_backup_20260412.sql
```

将备份文件传输到Linux服务器：
```bash
# 使用scp传输
scp ry_vue_backup_20260412.sql user@server-ip:/path/to/
```

#### 步骤B2：复制代码文件

传输整个项目到Linux服务器：
```bash
# 使用scp传输整个项目
scp -r /d/path/to/RuoYi-Vue user@server-ip:/opt/erp/
```

在Linux上目录结构：
```
/opt/erp/RuoYi-Vue/
├── ruoyi-admin/    # 后端代码
├── ruoyi-ui/       # 前端代码
└── pom.xml         # Maven配置
```

#### 步骤B3：安装环境（Ubuntu示例）

```bash
# 更新包列表
sudo apt update

# 1. 安装 MySQL 8.0
sudo apt install -y mysql-server
sudo mysql_secure_installation

# 2. 安装 JDK 17
sudo apt install -y openjdk-17-jdk

# 验证安装
java -version

# 3. 安装 Node.js 18
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node -v
npm -v

# 4. 安装 Maven
sudo apt install -y maven

# 验证安装
mvn -version
```

#### 步骤B4：导入数据库

```bash
# 连接MySQL
sudo mysql -u root -p
```

在MySQL命令行中执行：
```sql
CREATE DATABASE ry_vue DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'root'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON ry_vue.* TO 'root'@'%';
FLUSH PRIVILEGES;
EXIT;
```

导入数据：
```bash
mysql -u root -p ry_vue < /path/to/ry_vue_backup_20260412.sql
```

#### 步骤B5：配置数据库连接

修改配置文件：
```
/opt/erp/RuoYi-Vue/ruoyi-admin/src/main/resources/application-druid.yml
```

更新数据库连接信息：
```yaml
url: jdbc:mysql://localhost:3306/ry_vue?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
username: root
password: [你的MySQL密码]
```

#### 步骤B6：编译并启动后端

```bash
cd /opt/erp/RuoYi-Vue

# 编译后端
mvn clean install -DskipTests

# 启动后端（后台运行）
cd ruoyi-admin
nohup mvn spring-boot:run > spring-boot.log 2>&1 &
```

后端将运行在 `http://localhost:8080`

如果需要后台守护进程运行，可以使用 systemd 配置。

#### 步骤B7：安装依赖并启动前端

```bash
cd /opt/erp/RuoYi-Vue/ruoyi-ui

# 安装依赖
npm install

# 如果安装慢，可以使用淘宝镜像
# npm install --registry=https://registry.npmmirror.com

# 使用pmox启动前端（推荐）
sudo npm install -g pm2
pm2 start npm --name "erp-frontend" -- run dev

# 或者直接运行
# npm run dev
```

前端将运行在 `http://localhost:80`

如果需要外网访问，可以配置 Nginx 反向代理。

## 五、常见问题

### 5.1 后端启动失败 - Windows

**问题**：端口被占用
**解决**：
```powershell
# 查找占用8080端口的进程
netstat -ano | findstr :8080

# 杀死进程
taskkill /PID [进程ID] /F
```

### 5.2 后端启动失败 - Linux

**问题**：端口被占用
**解决**：
```bash
# 查找占用8080端口的进程
lsof -i :8080

# 杀死进程
kill -9 [进程ID]
```

### 5.3 数据库连接失败

**问题**：Access denied
**解决**：检查 `application-druid.yml` 中的数据库密码配置

### 5.4 Linux MySQL 远程连接失败

**问题**：无法从外部连接MySQL
**解决**：
```bash
# 编辑MySQL配置
sudo nano /etc/mysql/mysql.conf.d/mysqld.cnf

# 将 bind-address 改为 0.0.0.0
# bind-address = 0.0.0.0

# 重启MySQL
sudo systemctl restart mysql

# 确保用户权限正确
mysql -u root -p
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%' WITH GRANT OPTION;
FLUSH PRIVILEGES;
EXIT;
```

### 5.5 前端依赖安装失败

**问题**：npm install 报错
**解决**：
```bash
# 清除缓存
npm cache clean --force

# 使用淘宝镜像
npm install --registry=https://registry.npmmirror.com
```

### 5.6 Linux 防火墙问题

**问题**：无法访问前端/后端
**解决**：
```bash
# 开放端口
sudo ufw allow 80
sudo ufw allow 8080
sudo ufw reload
```

### 5.7 Maven 下载依赖慢

**问题**：Maven 编译很慢
**解决**：
配置阿里云镜像，修改 `settings.xml`：
```xml
<mirror>
  <id>aliyunmaven</id>
  <mirrorOf>*</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>
```
