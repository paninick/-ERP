# 系统整合架构设计方案

**设计日期**：2026-04-01  
**设计目标**：建立以专家团调用技能为核心驱动力、以MCP为执行中枢的系统整合架构

---

## 一、架构概述

### 1.1 设计原则
- **技能驱动**：以专家团调用技能为核心驱动力
- **MCP中枢**：以MCP（主控制程序）为执行中枢
- **模块化**：模块化设计，支持技能模块的无缝集成
- **可扩展**：支持未来新增技能模块
- **高可用**：确保系统响应时间≤500ms，技能调用成功率≥99.5%

### 1.2 系统组件
| 组件 | 职责 |
|------|------|
| **专家团** | 根据任务需求调用相应技能模块，制定技能匹配规则与优先级判定标准 |
| **MCP** | 任务分解、资源分配、进度监控、异常处理、结果整合 |
| **技能库** | 存储和管理各类技能模块 |
| **任务管理器** | 管理任务状态、跟踪进度、提供反馈 |

---

## 二、技能调用机制

### 2.1 技能分类
| 技能类别 | 技能描述 | 优先级 |
|---------|---------|--------|
| **搜索技能** | SearchCodebase、Grep、Glob、search（子代理） | P0 |
| **文件操作** | Read、Write、Edit、DeleteFile | P0 |
| **项目管理** | TodoWrite、TodoList | P1 |
| **系统工具** | RunCommand、CheckCommandStatus、StopCommand | P1 |
| **MCP工具** | run_mcp（调用各类MCP工具） | P0 |
| **Web工具** | OpenPreview、WebSearch、WebFetch | P2 |
| **诊断工具** | GetDiagnostics、AskUserQuestion | P2 |

### 2.2 技能匹配规则
```
技能匹配优先级：
1. 专用工具优先于通用工具
2. 子代理优先于单个工具
3. 并行执行优先于顺序执行
4. 搜索类：search（子代理）> SearchCodebase > Grep > Glob
5. 文件操作：优先使用Edit而非Write（除非创建新文件）
```

### 2.3 技能调用流程
1. 专家团接收任务
2. 分析任务需求，确定所需技能
3. 从技能库中查找匹配的技能
4. 按照优先级排序
5. 调用MCP执行技能
6. 获取结果并评估

---

## 三、MCP核心功能

### 3.1 任务分解
将复杂任务分解为可执行的子任务：
- 大任务→中等任务→小任务→原子操作
- 每个子任务分配明确的技能
- 建立任务间的依赖关系

### 3.2 资源分配
- 智能分配技能资源
- 避免资源冲突
- 支持并行执行
- 优先级调度

### 3.3 进度监控
- 实时跟踪任务状态
- 提供进度反馈
- 记录执行日志
- 超时检测

### 3.4 异常处理
- 捕获和处理技能调用异常
- 自动重试机制
- 降级处理策略
- 错误报告和恢复

### 3.5 结果整合
- 合并多个技能执行结果
- 格式化输出
- 结果验证
- 生成最终报告

## 四、接口规范

### 4.1 数据交互格式
**JSON格式**：
```json
{
  "taskId": "string",
  "taskType": "string",
  "priority": "P0|P1|P2",
  "skills": ["skill1", "skill2"],
  "parameters": {
    "key1": "value1",
    "key2": "value2"
  },
  "callbackUrl": "string"
}
```

### 4.2 通信协议
- **请求方式**：同步/异步
- **超时设置**：技能调用≤500ms
- **重试策略**：失败自动重试3次

### 4.3 错误码标准
| 错误码 | 含义 | 处理方式 |
|--------|------|---------|
| 200 | 成功 | 继续执行 |
| 400 | 参数错误 | 提示用户修正 |
| 404 | 技能未找到 | 使用替代技能 |
| 408 | 超时 | 重试或降级 |
| 500 | 内部错误 | 记录日志，通知管理员 |
