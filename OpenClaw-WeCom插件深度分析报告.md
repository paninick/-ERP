## 一、项目定位与核心价值

### 1.1 它是什么？

OpenClaw WeCom 插件是一个**企业微信多模式 AI 助手接入方案**，它将企业微信变成一个能长期工作的 AI 协作入口。

### 1.2 核心解决什么问题？

| 痛点 | 常规方案的问题 | OpenClaw 的解法 |
|------|--------------|----------------|
| **接入门槛高** | 需要公网IP+域名+SSL证书 | ✅ WebSocket长连接，**无需域名/IP** |
| **多人串上下文** | 多人共用机器人时互相污染对话 | ✅ **dynamicAgents 动态隔离**，每人每群独立Agent |
| **长任务超时丢失** | AI思考太久，企微连接断开结果发不回来 | ✅ **保活接力机制**，超时自动走备用通道投递 |
| **只能聊天不能干活** | AI只会在聊天框回复文字 | ✅ 接入**协作平面**（文档/日程/待办/通讯录） |
| **Bot vs Agent 二选一** | 要么体验好要么能力全 | ✅ **双模融合**：Bot WS负责实时交互 + Agent兜底正式推送 |

### 2.2 核心组件详解

#### 组件一：Bot WS（WebSocket 长连接）

| 特性 | 详情 | 对叫车业务的价值 |
|------|------|-----------------|
| **无需公网IP** | WebSocket出站连接，内网服务器即可部署 | ✅ 大幅降低部署门槛和成本 |
| **实时双向通信** | 收消息+发消息都在同一WebSocket通道 | ✅ 司机接单可即时确认反馈 |
| **流式回复支持** | 支持Markdown/混合内容流式输出 | ✅ 生成叫车清单时可逐步展示 |
| **媒体文件收发** | 图片/文件/语音自动处理 | ✅ 接收图片订单/发送Excel报表 |
| **心跳保活** | 自动重连（指数退避：1s→2s→4s→...→30s） | ✅ 7×24小时稳定运行不中断 |
| **混合消息接管** | 截图+文字等复合消息完整识别 | ✅ 用户发截图+文字指令都能处理 |

#### 组件二：Agent 模式（自建应用API）

| 特性 | 详情 | 对叫车业务的价值 |
|------|------|-----------------|
| **主动触达能力** | 不依赖已建立会话即可推送 | ✅ 定时发送每日叫车日报给指定群 |
| **组织级广播** | 按部门/标签批量发送 | ✅ 同时通知多个司机群/调度群 |
| **回调事件丰富** | 地理位置/交互卡片/菜单点击等 | ✅ 司机位置共享/一键接单卡片 |
| **权限体系完善** | 应用级权限控制 | ✅ 不同角色看到不同内容 |
| **正式投递稳定** | 走企业微信官方API通道 | ✅ 重要通知（如派单）不遗漏 |

#### 组件三：dynamicAgents（动态智能体隔离）

```
场景模拟：

  张三在「叫车调度A群」发了一条叫车需求
  → 系统自动创建 Agent: wecom-default-group-A123456-zhangsan
  → 该Agent独立保存张三的上下文和历史对话
  → 李四在同一群里发消息 → 创建另一个独立Agent
  → 两者完全隔离，不会串

  配置示例：
  {
    "channels": {
      "wecom": {
        "dynamicAgents": {
          "enabled": true,           // 开启动态隔离
          "dmCreateAgent": true,      // 私聊也隔离
          "groupEnabled": true,       // 群聊也隔离
          "adminUsers": ["manager01"] // 管理员绕过隔离
        }
      }
    }
  }
```

> **对叫车业务的意义**：多个调度员同时使用、多个司机群并行运作，绝对不会出现"张三的订单被李四看到"或"A群的消息发到B群"的情况。

## 三、安装与配置详解

### 3.1 快速安装（3步搞定）

```bash
# Step 1: 安装插件
openclaw plugins install @yanhaidao/wecom
openclaw plugins enable wecom

# Step 2: 互动向导配置（推荐新手）
openclaw channels add
# → 选择 "企业微信(WeCom)"
# → 输入 Bot ID 和 Secret

# Step 3: 启动服务
openclaw start
```

### 4.3 示例：叫车信息提取Tool代码框架

```python
# tools/extract_car_order.py

import re
from datetime import datetime
from openclaw.tools import BaseTool

class ExtractCarOrderTool(BaseTool):
    name = "extract_car_order"
    description = "从微信群消息中提取叫车订单关键信息"

    # 叫车信息提取规则库
    RULES = {
        'time': [
            r'(\d{1,2})月(\d{1,2})日\s*(\d{1,2}):?(\d{2})?',
            r'(明天|后天|今天)\s*(上午|下午)?\s*([一二三四五六七八九十]+)点',
            r'(\d{1,2}:\d{2})',
        ],
        'from_addr': [
            r'从(.+?)(?:到|去|往)',
            r'(.+?)出发',
            r'起点[：:]\s*(.+)',
        ],
        'to_addr': [
            r'到(.+?)(?:的|，|。|$)',
            r'去(.+?)(?:的|，|。|$)',
            r'终点[：:]\s*(.+)',
        ],
        'cargo_type': [
            r'(面料|辅料|成品|裁片|布料|纱线)',
        ],
        'cargo_qty': [
            r'(\d+(?:\.\d+)?)\s*(公斤|kg|斤|箱|包|件|吨)',
        ],
        'contact': [
            r'@(\S{2,4})',
            r'联系[\s:：]*(\S{2,4})',
        ],
        'phone': [
            r'1[3-9]\d{9}',
            r'(\d{3,4}[-\s]?\d{7,8})',
        ],
        'urgency': [
            r'(加急|紧急|ASAP|急|快点)',
        ],
    }

    def _execute(self, message_text: str, message_images=None):
        result = {
            'raw_message': message_text,
            'extracted_fields': {},
            'confidence': 100.0,
            'warnings': [],
            'status': 'success'
        }

        # 逐字段尝试匹配
        for field, patterns in self.RULES.items():
            matched = False
            for pattern in patterns:
                match = re.search(pattern, message_text)
                if match:
                    result['extracted_fields'][field] = match.group(
                        0 if field in ('cargo_type', 'urgency') else 
                        (-1 if len(match.groups()) == 0 else 1)
                    ).strip()
                    matched = True
                    break
            
            if not matched and field in ['time', 'from_addr', 'to_addr', 'contact']:
                result['confidence'] -= 10
                result['warnings'].append(f"未检测到{field}")

        # 置信度分级
        if result['confidence'] >= 95:
            result['auto_action'] = 'direct_import'
        elif result['confidence'] >= 80:
            result['auto_action'] = 'pending_review'
        else:
            result['auto_action'] = 'manual_required'

        return result
```

## 五、优势与局限性分析

### 5.1 核心优势

| 优势 | 详细说明 | 对项目的价值 |
|------|---------|------------|
| 🚀 **零门槛部署** | 无需公网IP、域名、SSL证书，内网即可运行 | 节省大量基础设施成本和时间 |
| 🔒 **会话硬隔离** | dynamicAgents确保多人/多群互不干扰 | 多调度员/多司机群安全并行 |
| ⏱️ **长任务保活** | 超时自动切换投递通道，复杂任务不丢结果 | 生成大型报表/分析不怕超时 |
| 🤖 **双模融合** | Bot WS体验丝滑 + Agent能力强大 | 日常交互和正式通知两不误 |
| 🔧 **高度可扩展** | 自定义Tool机制，任意扩展业务逻辑 | 叫车提取规则/图表生成/ERP对接全部可插拔 |
| 📱 **协作平面打通** | 直接操作企微文档/日程/待办/通讯录 | 叫车数据可直接写入协作文档 |
| 🔄 **自动重连** | 指数退避策略，断网恢复后自动重连 | 7×24无人值守稳定运行 |

### 5.2 局限性与应对方案
