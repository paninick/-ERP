# 专家团技能调用接口规范

**创建日期**：2026-04-01  
**版本**：1.0  
**用途**：定义专家团与MCP之间的接口规范

---

## 一、接口概述

### 1.1 设计原则
- **统一格式**：JSON格式的请求和响应
- **同步/异步**：支持同步和异步两种模式
- **错误处理**：统一的错误码和错误信息
- **可扩展**：支持新增技能模块

### 1.2 通信方式
专家团通过MCP调用技能，遵循以下流程：
1. 专家团解析任务，确定需要的技能
2. 专家团生成技能调用请求
3. MCP接收请求，调用相应技能
4. MCP返回技能执行结果
5. 专家团处理结果，决定下一步行动

## 二、请求接口规范

### 2.1 请求格式
```json
{
  "taskId": "string",
  "taskType": "SEARCH|FILE_OPERATION|PROJECT_MANAGEMENT|SYSTEM_TOOL|MCP_TOOL|DIAGNOSTIC|WEB",
  "priority": "P0|P1|P2",
  "skills": [
    {
      "name": "string",
      "parameters": {
        "key1": "value1",
        "key2": "value2"
      }
    }
  ],
  "executionMode": "SYNC|ASYNC",
  "callbackUrl": "string",
  "metadata": {
    "source": "string",
    "timestamp": "ISO8601",
    "retryCount": 0
  }
}
```

### 2.2 请求字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskId | string | 是 | 任务唯一标识 |
| taskType | string | 是 | 任务类型：SEARCH、FILE_OPERATION、PROJECT_MANAGEMENT、SYSTEM_TOOL、MCP_TOOL、DIAGNOSTIC、WEB |
| priority | string | 是 | 优先级：P0、P1、P2 |
| skills | array | 是 | 技能列表 |
| skills[].name | string | 是 | 技能名称 |
| skills[].parameters | object | 是 | 技能参数 |
| executionMode | string | 否 | 执行模式：SYNC（同步）、ASYNC（异步），默认SYNC |
| callbackUrl | string | 否 | 回调URL（异步模式时必填） |
| metadata | object | 否 | 元数据 |
| metadata.source | string | 否 | 来源标识 |
| metadata.timestamp | string | 否 | 请求时间戳 |
| metadata.retryCount | number | 否 | 重试次数 |

### 2.3 技能名称映射

| 技能类别 | 技能名称 |
|---------|---------|
| **搜索类** | SearchCodebase、Grep、Glob、search |
| **文件操作类** | Read、Write、Edit、DeleteFile |
| **项目管理类** | TodoWrite、TodoList |
| **系统工具类** | RunCommand、CheckCommandStatus、StopCommand |
| **诊断与交互类** | GetDiagnostics、AskUserQuestion、WebSearch、WebFetch |
| **Web预览类** | OpenPreview |
| **MCP工具** | run_mcp |

## 三、响应接口规范

### 3.1 成功响应格式
```json
{
  "taskId": "string",
  "status": "SUCCESS",
  "code": 200,
  "message": "string",
  "data": {
    "skill": "string",
    "result": "any",
    "executionTime": 0,
    "metadata": {
      "timestamp": "ISO8601"
    }
  },
  "metadata": {
    "timestamp": "ISO8601",
    "executionTime": 0
  }
}
```

### 3.2 失败响应格式
```json
{
  "taskId": "string",
  "status": "FAILED",
  "code": 400,
  "message": "string",
  "error": {
    "type": "string",
    "details": "string",
    "stackTrace": "string"
  },
  "metadata": {
    "timestamp": "ISO8601",
    "retryable": true
  }
}
```

### 3.3 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| taskId | string | 任务唯一标识 |
| status | string | 状态：SUCCESS、FAILED |
| code | number | 错误码：200（成功）、400（参数错误）、404（技能未找到）、408（超时）、500（内部错误） |
| message | string | 响应消息 |
| data | object | 成功时的数据（可选） |
| data.skill | string | 执行的技能名称 |
| data.result | any | 技能执行结果 |
| data.executionTime | number | 技能执行时间（毫秒） |
| error | object | 失败时的错误信息（可选） |
| error.type | string | 错误类型 |
| error.details | string | 错误详情 |
| error.stackTrace | string | 错误堆栈（可选） |
| metadata | object | 元数据 |
| metadata.timestamp | string | 响应时间戳 |
| metadata.executionTime | number | 总执行时间（毫秒） |
| metadata.retryable | boolean | 是否可重试 |

## 四、错误码标准

| 错误码 | 含义 | HTTP状态码 | 重试建议 |
|--------|------|-----------|---------|
| 200 | 成功 | 200 | 否 |
| 400 | 参数错误 | 400 | 否（需修正参数） |
| 404 | 技能未找到 | 404 | 否（使用替代技能） |
| 408 | 超时 | 408 | 是（最多重试3次） |
| 429 | 请求过多 | 429 | 是（等待后重试） |
| 500 | 内部错误 | 500 | 是（最多重试2次） |
