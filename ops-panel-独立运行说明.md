# ERP运维管理面板 - 独立运行说明

## 📋 概述

ERP运维管理面板是一个完全独立的运维工具，可以在所有其他服务（前端、后端、数据库等）关闭的情况下正常启动和运行。

## 🎯 核心特性

- ✅ **完全独立运行**：不依赖任何其他服务进程
- ✅ **实时服务监控**：监控后端、前端、MySQL、Redis状态
- ✅ **服务一键控制**：启动、停止、重启所有服务
- ✅ **问题自动诊断**：智能诊断并提供解决方案
- ✅ **Maven编译支持**：一键编译后端项目

## 🚀 快速启动

### 方式一：前台启动（推荐用于调试）

双击运行：
```
start-ops-panel.bat
```

特点：
- 显示实时日志
- 按 Ctrl+C 可停止
- 适合调试和查看输出

### 方式二：后台启动（推荐用于日常使用）

双击运行：
```
start-ops-panel-background.bat
```

特点：
- 在后台静默运行
- 不显示控制台窗口
- 适合长期运行

### 停止服务

双击运行：
```
stop-ops-panel.bat
```

## 🌐 访问地址

启动成功后，在浏览器中打开：

**http://localhost:9999**

## 🔧 技术架构

### 后端（Python Flask）

- **端口**：9999
- **依赖**：仅需 Python 3.7+ 和 Flask
- **功能**：
  - 服务状态监控
  - 服务启动/停止/重启
  - Maven编译
  - 日志查看
  - 问题诊断

### 前端（纯HTML/JS）

- **技术**：原生 HTML + JavaScript + CSS
- **特点**：无需构建，直接运行
- **通信**：通过 REST API 与后端交互

## ✅ 独立运行验证

### 验证步骤

1. **停止所有其他服务**
   - 停止后端（8080端口）
   - 停止前端（80端口）
   - 停止MySQL（3306端口）
   - 停止Redis（6379端口）

2. **启动运维面板**
   ```bash
   start-ops-panel.bat
   ```

3. **验证访问**
   - 打开浏览器访问 http://localhost:9999
   - 页面应正常显示
   - 可以看到所有服务状态为"未运行"

4. **功能测试**
   - 点击"诊断"按钮，应能正常工作
   - 可以查看服务状态
   - 即使其他服务未运行，面板本身完全可用

### 依赖检查

运维面板仅依赖：
- Python 3.7+
- Flask 库

**不依赖**：
- ❌ Java/JDK
- ❌ MySQL
- ❌ Redis
- ❌ Node.js
- ❌ 任何其他ERP服务

## 🛠️ 故障排查

### 问题：端口9999被占用

**解决方案**：
```bash
# 运行停止脚本
stop-ops-panel.bat

# 或手动查找并结束进程
netstat -ano | findstr ":9999"
taskkill /F /PID <进程ID>
```

### 问题：Python未找到

**解决方案**：
- 安装 Python 3.7 或更高版本
- 确保 Python 添加到系统 PATH
- 重启命令行窗口

### 问题：Flask未安装

**解决方案**：
```bash
pip install flask
```

## 📊 服务监控说明

运维面板监控以下服务：

| 服务 | 端口 | 说明 |
|------|------|------|
| 后端服务 | 8080 | Spring Boot 应用 |
| 前端服务 | 80 | Nginx 或 Vue dev server |
| MySQL | 3306 | 数据库服务 |
| Redis | 6379 | 缓存服务 |

**重要**：即使这些服务都未运行，运维面板本身仍可正常使用！
