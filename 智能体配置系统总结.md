## 一、系统概述

我们已经成功创建了一个完整的 AI Agent 配置系统，基于 `sanbuphy/learn-coding-agent` 项目的最佳实践。

### 核心特性

| 特性 | 说明 |
|------|------|
| 模块化设计 | 各配置模块独立，易于维护和扩展 |
| JSON 序列化 | 配置可保存为 JSON 文件，便于版本控制 |
| 预设配置 | 提供 MVP、生产、高级三种预设配置 |
| 12 层架构支持 | 完整支持 s01-s12 各层配置 |
| 工具配置 | 支持 10+ 个默认工具的详细配置 |

### 文件位置

- **配置系统代码**：`d:\erp\agent_config.py`
- **生成的配置文件**：`d:\erp\config\`

## 二、配置模块说明

### 2.1 ToolConfig（工具配置）

配置单个工具的详细信息：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `enabled` | 工具是否启用 | `true` |
| `concurrency_safe` | 是否并发安全 | `false` |
| `read_only` | 是否只读操作 | `false` |
| `destructive` | 是否破坏性操作 | `false` |
| `always_allow` | 是否始终允许 | `false` |
| `always_deny` | 是否始终拒绝 | `false` |
| `always_ask` | 是否始终询问 | `true` |
| `priority` | 工具优先级 | `0` |

**默认工具列表**：
1. `file_read` - 文件读取
2. `file_write` - 文件写入
3. `file_edit` - 文件编辑
4. `glob` - 文件搜索
5. `grep` - 内容搜索
6. `bash` - Bash 命令
7. `powerShell` - PowerShell 命令
8. `ask_user` - 用户询问
9. `web_fetch` - 网页获取
10. `web_search` - 网页搜索
11. `todo_write` - 任务列表
12. `mcp` - MCP 工具

### 2.2 FeatureConfig（功能特性配置）

配置 12 层架构及其他功能：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `s01_the_loop` | 基础循环 | `true` |
| `s02_tool_dispatch` | 工具调度 | `true` |
| `s03_planning` | 计划模式 | `true` |
| `s04_sub_agents` | 子 Agent | `false` |
| `s05_knowledge_on_demand` | 按需知识 | `false` |
| `s06_context_compression` | 上下文压缩 | `true` |
| `s07_persistent_tasks` | 持久化任务 | `true` |
| `s08_background_tasks` | 后台任务 | `false` |
| `s09_agent_teams` | Agent 团队 | `false` |
| `s10_team_protocols` | 团队协议 | `false` |
| `s11_autonomous_agents` | 自主 Agent | `false` |
| `s12_worktree_isolation` | 工作树隔离 | `false` |
| `auto_compact` | 自动压缩 | `true` |
| `snip_compact` | 裁剪压缩 | `false` |
| `context_collapse` | 上下文折叠 | `false` |
| `telemetry` | 遥测 | `false` |
| `undercover_mode` | 卧底模式 | `false` |

### 2.3 PersistenceConfig（持久化配置）

配置会话持久化：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `enabled` | 是否启用持久化 | `true` |
| `storage_dir` | 存储目录 | `"./agent_sessions"` |
| `jsonl_messages` | 使用 JSONL 存储消息 | `true` |
| `json_metadata` | 使用 JSON 存储元数据 | `true` |
| `fsync_user_messages` | 用户消息阻塞写入 | `true` |
| `async_other_messages` | 其他消息异步写入 | `true` |
| `auto_save_interval` | 自动保存间隔 | `10` |

### 2.4 CompressionConfig（压缩配置）

配置上下文压缩：

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `enabled` | 是否启用压缩 | `true` |
| `token_threshold` | Token 阈值 | `8000` |
| `max_messages` | 最大消息数 | `50` |
| `keep_recent` | 保留最近消息数 | `30` |
| `keep_oldest` | 保留最早消息数 | `10` |
| `compact_boundary_marker` | 压缩边界标记 | `"[COMPACT_BOUNDARY]"` |
| `use_llm_for_summary` | 使用 LLM 生成摘要 | `true` |
| `summary_model` | 摘要模型 | `"gpt-3.5-turbo"` |

### 2.5 PermissionConfig（权限配置）

| `always_ask_destructive` | 始终询问破坏性操作 | `true` |
| `interactive_prompt` | 交互式提示 | `true` |
| `remember_always_allow` | 记住始终允许 | `true` |
| `path_sandbox_enabled` | 路径沙箱启用 | `true` |
| `allowed_paths` | 允许的路径 | `[]` |

### 2.6 SystemPromptConfig（系统提示配置）

| 字段 | 说明 | 默认值 |

| `include_permissions` | 包含权限信息 | `true` |
| `include_project_context` | 包含项目上下文 | `true` |
| `include_claude_md` | 包含 CLAUDE.md | `true` |

## 四、使用方法

### 4.1 基本使用

```python
from agent_config import AgentConfig, get_mvp_config, get_production_config

# 创建默认配置
config = AgentConfig()

# 创建 MVP 配置
mvp_config = get_mvp_config()

# 创建生产配置
prod_config = get_production_config()

# 修改配置
config.name = "我的智能助手"
config.features.s04_sub_agents = True
config.compression.token_threshold = 10000

# 保存配置
config.save("./config/my_config.json")

# 加载配置
loaded_config = AgentConfig.load("./config/my_config.json")
```

### 4.2 运行示例

```bash
cd d:\erp
python agent_config.py
```

这将：
1. 创建默认配置
2. 创建 MVP 配置
3. 保存配置到 `./config/` 目录
4. 从文件加载配置
5. 创建并保存生产配置
