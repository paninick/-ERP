## 1. 测试目标

- 验证ERP系统各业务模块间接口调用和数据流转的正确性
- 验证跨模块业务流程的端到端功能完整性
- 验证数据一致性约束（库存、金额、状态同步）
- 验证系统在异常输入和边界条件下的健壮性
- 验证系统在并发负载下的性能指标达标

|------|------|

## 6. 数据一致性验证

### 6.1 订单状态同步验证

| 验证项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 订单状态与生产计划状态联动 | 订单确认后检查生产计划是否生成 | 生产计划自动创建，状态 "待排程" |
| 订单状态与出库数量联动 | 出库完成后检查订单已出库数量 | 已出库数量 = 出库单数量之和 |
| 订单完成条件 | 检查完成前各子流程是否全部完结 | 生产、质检、出库、财务全部完成后才可关闭 |
| 订单取消校验 | 尝试取消已开始生产的订单 | 系统拒绝，提示 "订单已在生产中，不可取消" |

### 6.2 库存数量一致性验证

| 验证项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 采购入库后库存 | 入库后查询库存台账 | 库存增加量 = 入库合格品数量 |
| 生产领料后库存 | 发料后查询库存台账 | 库存减少量 = 发料数量 |
| 销售出库后库存 | 出库后查询库存台账 | 成品库存减少量 = 出库数量 |
| 盘点后库存 | 盘点确认后查询库存 | 库存更新为实盘数量 |
| 库存流水完整性 | 查询库存流水表 | 每笔出入库均有对应流水记录，借贷平衡 |

### 6.3 财务金额平衡验证

| 验证项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 应收总额 = 销售出库总额 | 汇总已出库订单金额 | 金额一致 |
| 应付总额 = 采购入库总额 | 汇总已入库采购单金额 | 金额一致 |
| 收款后应收余额递减 | 收款后查询应收余额 | 应收余额 = 原应收 - 已收款 |
| 付款后应付余额递减 | 付款后查询应付余额 | 应付余额 = 原应付 - 已付款 |
| 发票金额不超过订单金额 | 开票时验证 | 系统校验，超限拒绝 |

### 6.4 跨模块数据追溯验证

| 验证项 | 验证方法 | 预期结果 |
|--------|----------|----------|
