## 团队协作规则

### 1. 代码规范

- 遵循阿里巴巴Java开发手册
- 使用统一的代码风格和规范
- 方法和变量命名要有意义
- 代码注释率不低于30%
- 避免使用魔法数字

### 2. 版本控制

- 使用Git进行版本控制
- 代码提交信息规范：`[模块名] 功能描述 [任务号]`
- 每个功能开发完成后创建Pull Request
- 代码审查通过后才能合并到主分支

### 3. 文档规范

- 需求文档：详细描述功能需求和业务流程
- 技术文档：系统架构、接口设计等
- 用户手册：系统使用说明
- 部署文档：服务器部署步骤

### 4. 沟通协作

- 每日站会：早上9:00，每个人汇报工作进度和问题
- 周例会：每周五下午，总结一周工作，规划下周计划
- 即时沟通：使用企业微信/钉钉进行实时沟通
- 项目管理工具：使用Jira或Trello管理任务和缺陷

## 项目执行流程

### 1. 开发流程

```
需求分析 → 系统设计 → 编码实现 → 代码审查 → 测试 → 部署
```

```
缺陷发现 → 缺陷记录 → 缺陷分配 → 缺陷修复 → 缺陷验证 → 缺陷关闭
```
