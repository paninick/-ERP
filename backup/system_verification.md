# 系统恢复后验证步骤

## 1. 服务状态验证

### 1.1 核心服务验证
1. **Redis服务**：
   - 检查Redis服务是否运行（端口6379）
   - 执行Redis命令验证连接：`redis-cli ping`
   - 预期结果：返回 `PONG`

2. **MySQL服务**：
   - 检查MySQL服务是否运行（端口3306）
   - 执行MySQL命令验证连接：`mysql -u root -p -e "SELECT 1;"`
   - 预期结果：返回 `1`

3. **后端服务**：
   - 检查后端服务是否运行（端口8080）
   - 执行HTTP请求验证API：`curl http://localhost:8080/ruoyi/login`
   - 预期结果：返回登录页面HTML

4. **前端服务**：
   - 检查前端服务是否运行（端口80）
   - 执行HTTP请求验证页面：`curl http://localhost/`
   - 预期结果：返回登录页面HTML

### 1.2 服务启动顺序验证
1. **验证服务启动顺序**：
   - Redis服务 → MySQL服务 → 后端服务 → 前端服务
   - 检查服务启动日志，确保启动顺序正确

## 2. 数据完整性验证

### 2.1 数据库验证
1. **数据库结构验证**：
   - 执行SQL查询验证表结构：`SHOW TABLES;`
   - 检查关键表是否存在：`sys_user`, `sys_role`, `sys_menu`, `sys_dict_type`

2. **数据库数据验证**：
   - 检查用户数据：`SELECT COUNT(*) FROM sys_user;`
   - 检查角色数据：`SELECT COUNT(*) FROM sys_role;`
   - 检查菜单数据：`SELECT COUNT(*) FROM sys_menu;`
   - 检查字典数据：`SELECT COUNT(*) FROM sys_dict_type;`

3. **关键数据验证**：
   - 检查管理员用户：`SELECT * FROM sys_user WHERE user_name = 'admin';`
   - 检查超级管理员角色：`SELECT * FROM sys_role WHERE role_key = 'admin';`

### 2.2 配置文件验证
1. **配置文件存在性验证**：
   - 检查后端配置文件：`D:\erp\RuoYi-Vue\ruoyi-admin\src\main\resources\application.yml`
   - 检查前端配置文件：`D:\erp\RuoYi-Vue\ruoyi-ui\.env.development`

2. **配置文件内容验证**：
   - 检查数据库连接配置
   - 检查Redis连接配置
   - 检查服务端口配置

### 2.3 上传文件验证
1. **上传目录验证**：
   - 检查上传目录是否存在：`D:\erp\RuoYi-Vue\ruoyi-admin\src\main\resources\static`
   - 检查上传文件是否完整
