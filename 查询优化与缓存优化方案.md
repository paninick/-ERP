## 一、查询优化方案

| 表名 | 推荐索引 | 说明 |

| demo_order | idx_factory_id, idx_order_no, idx_style_no, idx_order_date | 工厂ID、订单号、款号、订单日期 |

| demo_sales | idx_factory_id, idx_order_id, idx_sales_date | 工厂ID、订单ID、销售日期 |

### 1.2 查询SQL优化建议

**优化点1**：避免 SELECT *，只查询需要的字段
**优化点2**：使用 LIMIT 分页，避免全表扫描
**优化点3**：合理使用 JOIN，避免 N+1 查询
**优化点4**：使用 WHERE 条件过滤数据
**优化点5**：合理使用 ORDER BY，避免文件排序

### 1.3 MyBatis Plus 优化

**优化点1**：使用 LambdaQueryWrapper 构建查询
**优化点2**：合理使用 select 方法，指定字段
**优化点3**：使用 Page 分页对象
**优化点4**：合理使用缓存注解

## 二、缓存优化方案

### 2.1 缓存策略

| 缓存类型 | 缓存内容 | 缓存时间 | 说明 |
|---------|---------|---------|
| 用户信息缓存 | 用户权限、角色信息 | 30分钟 | 登录用户信息 |
| 字典数据缓存 | 系统字典数据 | 1小时 | 字典表数据 |
| 款式信息缓存 | 款式基本信息 | 10分钟 | 款式详情 |
| 订单统计缓存 | 订单统计数据 | 5分钟 | 统计报表 |
| 成本计算缓存 | 成本计算结果 | 3分钟 | 成本数据 |

### 2.2 Redis 缓存配置

**配置建议**：
- 缓存前缀：erp:demo:
- 缓存过期策略：LRU
- 内存限制：合理配置

### 2.3 缓存更新策略

**策略1**：主动更新 - 数据变更时立即更新缓存
**策略2**：被动更新 - 缓存过期后自动重新加载
**策略3**：混合策略 - 重要数据主动更新，次要数据被动更新
