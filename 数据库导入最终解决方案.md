# 数据库导入最终解决方案

## 问题
由于 PowerShell 与 MySQL 命令行的兼容性问题，无法通过脚本自动导入 SQL 文件。

## 解决方案（手动操作）

### 方法一：使用 MySQL 命令行（最简单）

1. **打开命令提示符（CMD）**
   - 按 `Win + R`
   - 输入 `cmd`
   - 按回车

2. **连接到 MySQL**
   ```cmd
   cd C:\Program Files\MySQL\MySQL Server 8.4\bin
   mysql -u root -p
   ```
   输入密码：`password`

3. **在 MySQL 提示符下执行**
   ```sql
   USE `ry-vue`;
   SOURCE d:/erp/RuoYi-Vue/sql/ry_20260321.sql;
   ```

4. **验证导入成功**
   ```sql
   SHOW TABLES;
   SELECT menu_id, menu_name FROM sys_menu WHERE menu_id <= 4;
   EXIT;
   ```

### 方法二：使用 MySQL Workbench（如果已安装）

1. 打开 MySQL Workbench
2. 连接到本地 MySQL（root/password）
3. 选择 `ry-vue` 数据库
4. 打开 SQL 文件：`d:\erp\RuoYi-Vue\sql\ry_20260321.sql`
5. 点击闪电图标执行

### 方法三：使用 Navicat 或其他数据库工具

1. 连接到 MySQL
2. 选择 `ry-vue` 数据库
3. 右键 → 运行 SQL 文件
4. 选择 `d:\erp\RuoYi-Vue\sql\ry_20260321.sql`
5. 执行

## 导入成功后

1. **确认后端已停止**
2. **重新启动后端应用**
   ```cmd
   cd d:\erp\RuoYi-Vue\ruoyi-admin
   java -jar target\ruoyi-admin.jar
   ```

3. **前端已经在运行**（http://localhost:80/）

4. **访问系统**
   - 打开浏览器访问 http://localhost:80/
   - 登录账号：admin
   - 登录密码：admin123
