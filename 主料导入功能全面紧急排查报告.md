## 四、潜在问题分析

### 问题1: Excel列名与@Excel注解不匹配
**可能性**: ⭐⭐⭐⭐⭐ (高)

**症状**:
- Excel读取到的数据为空或部分字段为空
- 日志中显示读取到0条数据或字段值为null

**排查方法**:
1. 查看Controller日志中"读取到第X条数据"的输出
2. 确认Excel文件的表头列名是否与 `@Excel(name="xxx")` 完全一致
3. 特别注意空格、特殊字符

**预期Excel表头列名**:
```
主料类型 | 主料编号 | 供货方式 | 厂供编号 | 主料供应商id | 主料品名 | 主料成分 | 主料门幅 | 主料克重 | 主料纱支 | 计量单位 | 图片
```

---

### 问题2: 数据库缺少 factory_no 字段
**可能性**: ⭐⭐⭐⭐ (中高)

**症状**:
- 导入时报错：`Unknown column 'factory_no' in 'field list'`
- 数据插入失败

**排查方法**:
1. 执行SQL: `DESCRIBE t_erp_main_material;`
2. 查看是否存在 `factory_no` 字段

**修复方法**:
```sql
ALTER TABLE t_erp_main_material 
ADD COLUMN factory_no VARCHAR(100) DEFAULT NULL COMMENT '厂供编号' 
AFTER supply_method;
```

---

### 问题3: supplierId 类型转换问题
**可能性**: ⭐⭐⭐ (中)

**症状**:
- 供应商ID字段读取失败
- 报错：`NumberFormatException`

**原因**:
- `supplierId` 是 `Long` 类型
- Excel中该列可能是空值或非数字内容

**建议**:
- 确保Excel中"主料供应商id"列是数字或留空
- 留空时会自动处理为null

### 第一步：查看日志
启动应用后，执行导入操作，查看日志输出：

```
========== 接收到主料导入请求 ==========
文件名: xxx.xlsx, 文件大小: xxx bytes
是否支持更新: false
开始读取Excel文件...
Excel读取完成，读取到 X 条数据
读取到第 1 条数据: MainMaterial(...)
========== 开始导入主料数据 ==========
接收数据数量: X
处理第 1 条数据: ...
...
========== 主料数据导入完成 ==========
成功: X 条, 失败: X 条
```

### 第二步：根据日志判断问题

| 日志现象 | 可能原因 | 排查方向 |
|---------|---------|---------|
| "Excel读取完成，读取到 0 条数据" | Excel文件格式问题、表头不匹配 | 检查Excel表头列名 |
| 读取到数据但字段全为null | 列名与@Excel不匹配 | 对比列名与注解 |
| factory_no字段为null但Excel有值 | 列名不对 或 数据库无此字段 | 检查列名和数据库表结构 |
| 插入时报错"Unknown column" | 数据库缺少字段 | 执行SQL添加字段 |

### 第三步：验证数据库表结构
```sql
-- 1. 查看表结构
DESCRIBE t_erp_main_material;

-- 2. 如果factory_no不存在，执行
ALTER TABLE t_erp_main_material 
ADD COLUMN factory_no VARCHAR(100) DEFAULT NULL COMMENT '厂供编号' 
AFTER supply_method;

-- 3. 再次验证
DESCRIBE t_erp_main_material;
```
