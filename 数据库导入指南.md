# 数据库导入指南

## 问题说明
由于 PowerShell 与 MySQL 命令行的兼容性问题，无法通过脚本自动导入 SQL 文件。需要手动导入。

## 方法一：使用 MySQL 命令行（推荐）

### 步骤：

1. **打开命令提示符（CMD）或 PowerShell**

2. **进入 MySQL bin 目录**
   ```cmd
   cd C:\Program Files\MySQL\MySQL Server 8.4\bin
   ```

3. **连接到 MySQL**
   ```cmd
   mysql -u root -p
   ```
   输入密码：`password`

4. **在 MySQL 提示符下执行以下命令**
   ```sql
   USE `ry-vue`;
   SOURCE d:/erp/RuoYi-Vue/sql/ry_20260321.sql;
   ```

5. **等待导入完成**（约 62KB，很快）

6. **验证导入成功**
   ```sql
   SHOW TABLES;
   SELECT COUNT(*) FROM sys_menu;
   SELECT menu_id, menu_name FROM sys_menu WHERE menu_id &lt;= 4;
   ```

7. **退出 MySQL**
   ```sql
   EXIT;
   ```

## 方法二：使用 MySQL Workbench

如果安装了 MySQL Workbench：

1. 打开 MySQL Workbench
2. 连接到本地 MySQL（root/password）
3. 打开 SQL 编辑器
4. 执行：`USE \`ry-vue\`;`
5. 打开文件：`d:\erp\RuoYi-Vue\sql\ry_20260321.sql`
6. 执行整个 SQL 文件

## 方法三：使用 Navicat 或其他数据库工具

1. 连接到 MySQL
2. 选择 `ry-vue` 数据库
3. 右键 → 运行 SQL 文件
4. 选择 `d:\erp\RuoYi-Vue\sql\ry_20260321.sql`
5. 执行

## 导入成功后

1. **确认后端已停止**（如果正在运行）
2. **重新启动后端应用**
   ```cmd
   cd d:\erp\RuoYi-Vue\ruoyi-admin
   java -jar target\ruoyi-admin.jar
   ```

3. **前端应该已经在运行**（http://localhost:80/）

4. **访问系统并验证菜单显示**

## 当前系统状态

- ✅ MySQL：运行中（3306 端口）
- ✅ 前端：运行中（80 端口）
- ❌ 后端：需要重启（缺少表）
- ❌ 数据库：需要导入完整数据
