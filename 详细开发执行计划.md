## 四、技术规范与架构设计

### 4.1 代码规范

| 规范项 | 要求 |
|--------|------|
| 命名规范 | 遵循阿里巴巴 Java 开发手册 |
| 注释规范 | 所有类和方法必须有 Javadoc 注释 |
| 格式规范 | 使用若依框架代码风格 |
| 提交规范 | 每个功能完成后立即提交 |

### 4.2 数据验证规范

| 验证类型 | 注解 | 应用场景 |
|---------|------|---------|
| 非空验证 | `@NotNull` | 必填字段 |
| 字符串非空 | `@NotBlank` | 字符串字段 |
| 长度验证 | `@Size` | 字符串字段 |
| 数值范围 | `@Min`/`@Max` | 数字字段 |
| 金额精度 | `@Digits` | BigDecimal字段 |
| 数值范围 | `@DecimalMin`/`@DecimalMax` | 数值字段 |

### 4.3 异常处理规范

| 异常类型 | 使用场景 |
|---------|---------|
| 业务异常 | 业务逻辑错误 |
| 参数异常 | 参数验证失败 |
| 权限异常 | 权限不足 |
| 数据异常 | 数据不存在或无效 |

### 4.4 测试规范

| 测试类型 | 覆盖要求 |
|---------|---------|
| 单元测试 | 核心模块 ≥ 60% |
| 集成测试 | 核心场景 ≥ 50% |
| 命名规范 | test[FeatureName |
