# 任务状态跟踪与反馈机制

**创建日期**：2026-04-01  
**版本**：1.0  
**用途**：建立任务状态跟踪与反馈机制，确保任务执行过程可追踪、可监控

---

## 一、任务状态定义

### 1.1 任务生命周期状态

| 状态 | 描述 | 下一状态 |
|------|------|---------|
| **PENDING** | 任务已创建，等待执行 | RUNNING |
| **RUNNING** | 任务正在执行中 | COMPLETED、FAILED、PAUSED |
| **PAUSED** | 任务已暂停 | RUNNING |
| **COMPLETED** | 任务执行成功 | -（结束状态） |
| **FAILED** | 任务执行失败 | -（结束状态） |
| **CANCELLED** | 任务已取消 | -（结束状态） |

### 1.2 任务进度状态

| 进度 | 描述 | 说明 |
|------|------|------|
| 0% | 未开始 | 任务刚创建，尚未开始 |
| 0-25% | 准备中 | 正在准备执行环境 |
| 25-50% | 执行中 | 正在执行主要任务 |
| 50-75% | 处理中 | 正在处理结果 |
| 75-100% | 完成中 | 正在收尾工作 |
| 100% | 已完成 | 任务执行完毕 |

## 二、任务数据结构

### 2.1 任务对象定义

```json
{
  "taskId": "TASK-YYYYMMDD-XXXX",
  "taskName": "string",
  "taskType": "SEARCH|FILE_OPERATION|PROJECT_MANAGEMENT|SYSTEM_TOOL|MCP_TOOL|DIAGNOSTIC|WEB",
  "priority": "P0|P1|P2",
  "status": "PENDING|RUNNING|PAUSED|COMPLETED|FAILED|CANCELLED",
  "progress": 0,
  "skills": [
    {
      "skillId": "string",
      "skillName": "string",
      "parameters": {},
      "status": "PENDING|RUNNING|COMPLETED|FAILED",
      "result": {},
      "error": {}
    }
  ],
  "createdAt": "ISO8601",
  "startedAt": "ISO8601",
  "completedAt": "ISO8601",
  "estimatedDuration": 0,
  "actualDuration": 0,
  "createdBy": "string",
  "metadata": {},
  "result": {},
  "error": {}
}
```

### 2.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| taskId | string | 是 | 任务唯一标识，格式：TASK-YYYYMMDD-XXXX |
| taskName | string | 是 | 任务名称 |
| taskType | string | 是 | 任务类型 |
| priority | string | 是 | 优先级 |
| status | string | 是 | 任务状态 |
| progress | number | 是 | 进度百分比（0-100） |
| skills | array | 是 | 技能列表 |
| skills[].skillId | string | 是 | 技能唯一标识 |
| skills[].skillName | string | 是 | 技能名称 |
| skills[].parameters | object | 是 | 技能参数 |
| skills[].status | string | 是 | 技能状态 |
| skills[].result | object | 否 | 技能执行结果 |
| skills[].error | object | 否 | 技能执行错误 |
| createdAt | string | 是 | 创建时间 |
| startedAt | string | 否 | 开始时间 |
| completedAt | string | 否 | 完成时间 |
| estimatedDuration | number | 否 | 预估执行时间（毫秒） |
| actualDuration | number | 否 | 实际执行时间（毫秒） |
| createdBy | string | 是 | 创建者 |
| metadata | object | 否 | 元数据 |
| result | object | 否 | 任务结果 |
| error | object | 否 | 任务错误 |

## 三、状态变更流程

### 3.1 状态流转图

```
PENDING → RUNNING → COMPLETED
    ↓         ↓
CANCELLED  FAILED
              ↑
           PAUSED
```

### 3.2 状态变更规则

| 当前状态 | 允许的新状态 | 触发条件 |
|---------|------------|---------|
| PENDING | RUNNING, CANCELLED | 开始执行任务 / 取消任务 |
| RUNNING | COMPLETED, FAILED, PAUSED, CANCELLED | 执行成功 / 执行失败 / 暂停执行 / 取消任务 |
| PAUSED | RUNNING, CANCELLED | 恢复执行 / 取消任务 |
| COMPLETED | - | 结束状态，不可变更 |
| FAILED | - | 结束状态，不可变更 |
| CANCELLED | - | 结束状态，不可变更 |

## 四、反馈机制

### 4.1 进度反馈

**反馈频率**：
- P0优先级任务：每10%进度反馈一次
- P1优先级任务：每25%进度反馈一次
- P2优先级任务：每50%进度反馈一次

**反馈内容**：
```json
{
  "taskId": "string",
  "status": "RUNNING",
  "progress": 50,
  "message": "string",
  "currentSkill": "string",
  "estimatedRemaining": 0,
  "timestamp": "ISO8601"
}
```

### 4.2 完成反馈

**成功完成**：
```json
{
  "taskId": "string",
  "status": "COMPLETED",
  "progress": 100,
  "message": "Task completed successfully",
  "result": {},
  "actualDuration": 0,
  "timestamp": "ISO8601"
}
```

**失败完成**：
```json
{
  "taskId": "string",
  "status": "FAILED",
  "progress": 0,
  "message": "Task failed",
  "error": {
    "type": "string",
    "message": "string",
    "details": "string",
    "retryable": true
  },
  "actualDuration": 0,
  "timestamp": "ISO8601"
}
```

### 4.3 异常告警

**告警级别**：
- **ERROR**：任务执行失败，需要立即处理
- **WARNING**：任务执行有风险，但可以继续
- **INFO**：一般信息，不需要特殊处理

**告警内容**：
```json
{
  "taskId": "string",
  "level": "ERROR|WARNING|INFO",
  "message": "string",
  "details": "string",
  "actionRequired": true,
  "timestamp": "ISO8601"
}
```

## 五、任务监控面板

### 5.1 监控指标

| 指标 | 说明 | 更新频率 |
|------|------|---------|
| 总任务数 | 所有任务的总数 | 实时 |
| 待处理任务数 | PENDING状态的任务数 | 实时 |
| 执行中任务数 | RUNNING状态的任务数 | 实时 |
| 已完成任务数 | COMPLETED状态的任务数 | 实时 |
| 失败任务数 | FAILED状态的任务数 | 实时 |

## 七、重试机制

### 7.1 重试策略

| 错误类型 | 重试次数 | 重试间隔 |
|---------|---------|---------|
| 超时（408） | 3次 | 1s, 2s, 4s |
| 请求过多（429） | 3次 | 2s, 4s, 8s |
| 内部错误（500） | 2次 | 1s, 2s |

### 7.2 退避算法

使用指数退避算法：
```
重试间隔 = 初始间隔 × (2 ^ 重试次数)
```

| 任务响应时间 | ≤500ms | 从创建到开始执行的时间 |

| 可用性 | ≥99.9% | 系统正常运行时间占比 |
