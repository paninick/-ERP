## 1. 系统概述

### 1.1 设计目标
- 基于现有若依（RuoYi）框架扩展
- 实现实时监控、服务控制、数据备份三大核心功能
- 支持高并发、低延迟的实时数据推送
- 提供直观易用的用户界面
- 确保数据安全和操作可审计

### 1.2 设计原则
- **模块化**：功能模块独立，易于扩展和维护
- **可扩展性**：支持水平扩展，应对未来增长
- **高可用性**：系统本身具备高可用特性
- **安全性**：权限控制、数据加密、操作审计
- **用户友好**：直观易用，技术小白也能操作

### 2.2 分层架构

#### 2.2.1 前端层
- **框架**：Vue 2.x + Element UI
- **图表**：ECharts 5.x
- **状态管理**：Vuex
- **路由**：Vue Router
- **实时通信**：WebSocket客户端

#### 2.2.2 后端层
- **框架**：Spring Boot 2.7.x
- **语言**：Java 17
- **API风格**：RESTful + WebSocket
- **实时推送**：Spring WebSocket
- **系统监控**：OSHI (Java)
- **任务调度**：Quartz

#### 2.2.3 数据层
- **关系型数据库**：MySQL 8.0
- **缓存**：Redis 6.x
- **消息队列**：Redis List（可选）

### 4.2 核心表结构

#### 4.2.1 服务表 (ops_service)
```sql
CREATE TABLE ops_service (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL COMMENT '服务名称',
    type VARCHAR(50) NOT NULL COMMENT '服务类型',
    status VARCHAR(20) NOT NULL DEFAULT 'unknown' COMMENT '状态',
    health_check_url VARCHAR(500) COMMENT '健康检查URL',
    host VARCHAR(100) COMMENT '主机',
    port INT COMMENT '端口',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_status (status),
    INDEX idx_type (type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='服务表';
```

#### 4.2.2 备份记录表 (ops_backup_record)
```sql
CREATE TABLE ops_backup_record (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    service_id BIGINT COMMENT '关联服务ID',
    type VARCHAR(50) NOT NULL COMMENT '备份类型',
    scope JSON COMMENT '备份范围',
    status VARCHAR(20) NOT NULL COMMENT '状态',
    file_path VARCHAR(500) COMMENT '文件路径',
    file_size BIGINT COMMENT '文件大小(字节)',
    checksum_md5 VARCHAR(32) COMMENT 'MD5校验',
    checksum_sha256 VARCHAR(64) COMMENT 'SHA256校验',
    started_at DATETIME COMMENT '开始时间',
    completed_at DATETIME COMMENT '完成时间',
    error_message TEXT COMMENT '错误信息',
    created_by VARCHAR(100) COMMENT '创建人',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_service_id (service_id),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='备份记录表';
```

#### 4.2.3 定时备份配置表 (ops_scheduled_backup)
```sql
CREATE TABLE ops_scheduled_backup (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    service_id BIGINT COMMENT '关联服务ID',
    name VARCHAR(100) NOT NULL COMMENT '配置名称',
    schedule_type VARCHAR(20) NOT NULL COMMENT '周期类型',
    schedule_time TIME NOT NULL COMMENT '备份时间',
    retention_policy VARCHAR(20) NOT NULL COMMENT '保留策略',
    retention_count INT COMMENT '保留数量',
    retention_days INT COMMENT '保留天数',
    enabled BOOLEAN NOT NULL DEFAULT TRUE COMMENT '是否启用',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_service_id (service_id),
    INDEX idx_enabled (enabled)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='定时备份配置表';
```

#### 4.2.4 告警历史表 (ops_alert_history)
```sql
CREATE TABLE ops_alert_history (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    service_id BIGINT COMMENT '关联服务ID',
    backup_record_id BIGINT COMMENT '关联备份ID',
    alert_type VARCHAR(50) NOT NULL COMMENT '告警类型',
    level VARCHAR(20) NOT NULL COMMENT '告警级别',
    title VARCHAR(200) NOT NULL COMMENT '告警标题',
    message TEXT COMMENT '告警内容',
    metric_value DECIMAL(20,4) COMMENT '指标值',
    threshold_value DECIMAL(20,4) COMMENT '阈值',
    resolved BOOLEAN NOT NULL DEFAULT FALSE COMMENT '是否已解决',
    resolved_at DATETIME COMMENT '解决时间',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_service_id (service_id),
    INDEX idx_level (level),
    INDEX idx_created_at (created_at),
    INDEX idx_resolved (resolved)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='告警历史表';
```

#### 4.2.5 操作日志表 (ops_operation_log)
