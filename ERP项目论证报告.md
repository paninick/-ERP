## 一、项目概述

### 1.1 项目背景

富泉服装作为一家拥有多个生产工厂的针织服装企业，目前面临以下痛点：

| 痛点 | 现状描述 | 影响 |
|------|---------|------|
| **数据分散** | 各工厂独立使用Excel管理业务 | 无法实时汇总，决策滞后 |
| **库存积压** | 库存信息不透明，积压严重 | 资金占用成本高 |
| **成本不准** | FOB报价和成本核算依赖经验 | 报价失误导致亏损 |
| **沟通低效** | 跨工厂协同靠微信/电话 | 信息传递易出错 |
| **追溯困难** | 订单进度、质量问题难追溯 | 客户满意度下降 |

### 1.2 项目目标

| 目标维度 | 具体指标 |
|---------|---------|
| **上线工厂** | 2-3个核心生产工厂 |
| **覆盖模块** | 订单、生产、库存、采购、销售、财务 |
| **数据实时** | 业务数据实时同步，跨工厂可见 |
| **效率提升** | 业务流程效率提升30%以上 |
| **成本降低** | 库存资金占用降低20%以上 |

### 1.3 技术方案摘要

| 项 | 方案选择 |
|---|----------|
| **开发框架** | RuoYi-Vue 前后端分离 |
| **权限框架** | Apache Shiro（而非Spring Security） |
| **前端技术** | Vue 3 + Element Plus |
| **后端技术** | Spring Boot 2.x + MyBatis |
| **数据库** | MySQL 5.7+ |
| **缓存** | Redis |
| **部署环境** | 阿里云ECS + RDS + Redis |

#### 2.1.2 多工厂数据隔离方案验证

| 方案选项 | 优点 | 缺点 | 推荐度 |
|---------|------|------|--------|
| **软隔离（factory_id字段）** | 部署简单、跨工厂统计方便、扩展性好 | 单库压力集中（初期可接受） | ⭐⭐⭐⭐⭐ 强烈推荐 |
| **分库隔离** | 数据物理隔离、性能好 | 跨库查询复杂、运维成本高 | ⭐⭐ 不推荐初期 |
| **分库分表** | 扩展性最强 | 实现复杂度最高、维护成本高 | ⭐ 不推荐初期 |

**方案验证结论**：✅ **选择软隔离方案，合理且可行，符合当前规模**

### 3.4 业务流程优化分析

#### 3.4.1 当前业务流程痛点

| 流程 | 当前方式 | 痛点 | 影响 |
|------|---------|------|------|
| **订单报价** | Excel计算，人工汇总 | 慢、不准、易出错 | 报价失误导致亏损 |
| **生产排程** | Excel排程，微信群通知 | 信息分散，难跟踪 | 延期、漏单 |
| **库存管理** | 手工记账，定期盘点 | 不实时，账实不符 | 积压或缺货 |
| **成本核算** | 月底手工统计 | 滞后，不准确 | 决策无数据支撑 |
| **跨工厂协同** | 电话/微信沟通 | 信息传递易出错 | 效率低、责任不清 |

#### 3.4.2 ERP系统带来的流程优化

| 流程 | 优化后方式 | 效果 | 提升 |
