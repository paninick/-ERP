## 🚀 前端启动步骤

### 步骤 1：进入前端项目目录

```bash
cd d:\erp\RuoYi-Vue\ruoyi-ui
```

### 步骤 2：安装前端依赖

```bash
npm install
```

**注意**：
- 首次安装可能需要 5-10 分钟
- 如果安装失败，可以尝试使用国内镜像：
  ```bash
  npm install --registry=https://registry.npmmirror.com
  ```

### 步骤 3：启动前端开发服务器

```bash
npm run dev
```

**默认访问地址**：
- 前端：http://localhost:80
- 后端 API：http://localhost:8080

## 📝 前端配置

### 检查 .env.development 配置

文件位置：`d:\erp\RuoYi-Vue\ruoyi-ui\.env.development`

```env
# 若依管理系统/开发环境
VUE_APP_BASE_API = '/dev-api'
```

### 检查 vue.config.js 代理配置

文件位置：`d:\erp\RuoYi-Vue\ruoyi-ui\vue.config.js`

默认代理配置应该是：
```javascript
devServer: {
  host: '0.0.0.0',
  port: 80,
  open: true,
  proxy: {
    [process.env.VUE_APP_BASE_API]: {
      target: `http://localhost:8080`,
      changeOrigin: true,
      pathRewrite: {
        ['^' + process.env.VUE_APP_BASE_API]: ''
      }
    }
  }
}
```

## ⚠️ 常见问题

### 问题1：npm install 失败

**解决方案**：
```bash
# 清理缓存
npm cache clean --force

# 使用国内镜像
npm install --registry=https://registry.npmmirror.com
```

### 问题2：端口 80 被占用

**解决方案**：
1. 检查占用端口的进程：
   ```bash
   netstat -ano | findstr :80
   ```
2. 修改 vue.config.js 中的端口为其他端口（如 8081）

### 问题3：前端无法连接后端

**检查清单**：
- [ ] 后端应用是否正在运行（8080 端口）
- [ ] vue.config.js 中的代理配置是否正确
- [ ] 浏览器控制台是否有错误信息
